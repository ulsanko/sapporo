# 2026 삿포로 HCI 데이터 해커톤 — 제공 데이터셋

여름 삿포로의 문제(폭염·곰·관광)를 데이터로 다루기 위한 ML급 데이터 4종입니다.
모두 일본 공식 출처의 원본을 정제했습니다.

## 다운로드 방법

- **전체 받기:** 이 페이지 위쪽 초록색 **`Code` 버튼 → Download ZIP** (한 번에 4개 + 설명서).
- **개별 받기:** 아래 파일명을 클릭 → 오른쪽 위 **download(↓) 버튼**.
- 받은 뒤 압축을 풀고, 파이썬(pandas)·엑셀 등에서 바로 엽니다. 인코딩은 UTF-8(BOM)이라 한글·일본어가 깨지지 않습니다.

## 전처리부터가 챌린지

원본이 일본 공식 자료라 **일본어·합계행·결측**이 섞여 있습니다. 모델에 넣기 전에 정리하세요. — *정돈된 데이터(Tidy Data) 3원칙: ① 한 열 = 한 변수 · ② 한 행 = 한 관측 · ③ 한 칸 = 한 값*

**일본어 값 빠른 해석** (형식: 일본어(발음) — 뜻)

- 기상: `真夏日`(마나쓰비) 한여름날·최고 30℃↑ · `猛暑日`(모-쇼비) 폭염일·35℃↑ · `熱帯夜`(넷타이야) 열대야·최저 25℃↑
- 곰: `ヒグマ`(히구마) 불곰 · `目撃`(모쿠게키) 목격 · `フン`(훈) 배설물 · `足跡`(아시아토) 발자국 · `親子`(오야코) 어미와 새끼 · `南区`(미나미쿠) 남구(출몰의 71%)
- 관광: `韓国`/`中国`/`台湾`/`香港` 한·중·대·홍콩 · `米国` 미국 · `豪州` 호주 · `その他`(소노타) 기타

> ⚠️ **합계행 주의:** 관광 데이터의 `総数`(전체 합계)·`アジア計`·`ヨーロッパ計` 등 `〜計`(소계)는 합계행입니다. **국가별 분석 시 반드시 제외**(안 그러면 두 번 셉니다).
> 열중증 발생장소(`place_*`) 항목은 **2019년 이전이 빈칸**(결측)입니다.
